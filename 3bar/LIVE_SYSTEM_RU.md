# 3bar Live

## Что это

Это автономный live-пакет для системы `3bar` в отдельной папке.

Он запускается в своем собственном окне терминала и использует свои отдельные:

- `state/`
- `logs/`
- `reports/live/`
- lock-файл `state/three_bar_system.lock`

Это нужно, чтобы система не мешала другим ботам в проекте.

## Как запустить

### По двойному клику

- двойной клик по `launch_3bar_window.cmd`

Откроется отдельное окно PowerShell и внутри запустится daemon в paper-режиме.

### Paper launch

```powershell
.\3bar\start_3bar_paper.ps1
```

В paper-режиме система теперь ведет локальную бумажную позицию:

- хранит активную paper-сделку в `state/live_state.json`
- сопровождает paper-стоп и paper-trailing по тем же правилам
- записывает закрытые paper-сделки в `reports/live/paper_trades.jsonl`

### Real launch

```powershell
.\3bar\start_3bar_real.ps1
```

Для real-режима нужно:

- заполнить `.env`
- поставить `PAPER=false`
- иметь ключи BingX

## Как работает система

Сигнал:

- цена у локального уровня
- первая свеча - пин-бар
- вторая свеча - разноцветное поглощение тела первой
- третья свеча подтверждает движение и дает вход

Фильтры:

- `long` берется только по EMA-тренду вверх
- `short` остается без трендового фильтра

Выход:

- стартовый стоп сразу за тенью пин-свечи
- после `1.4R` включается трейлинг
- трейлинг идет по экстремумам последних `4` закрытых часовиков
- максимальное удержание `48` часов

## Что делает live daemon

- регулярно качает свежие `1h` свечи ETH с BingX
- ищет свежий сигнал
- не повторяет уже исполненный сигнал
- не открывает новую сделку, если уже есть позиция или открытые ордера
- открывает market-вход
- ставит стартовую защиту
- дальше в цикле подтягивает стоп по правилам стратегии
- при превышении максимального времени удержания закрывает позицию по рынку

В `PAPER=true`:

- реальные биржевые ордера не отправляются
- но локальная paper-позиция открывается, сопровождается и закрывается внутри `3bar`
- это позволяет проверять live-пайплайн и paper-PnL без риска для счета

## Какие файлы главные

- `launch_3bar_window.cmd` - запуск двойным кликом в отдельном окне
- `run_3bar_system.ps1` - окно daemon по умолчанию
- `run_3bar_daemon.ps1` - PowerShell runner daemon
- `three_bar_system_daemon.py` - daemon с lock-защитой
- `three_bar_daemon.py` - постоянный live-цикл
- `three_bar_live.py` - scan / execute / trailing logic
- `paper_summary.ps1` - сводка по paper-сделкам, активной paper-позиции и суммарному paper-PnL
- `.env.example` - шаблон настроек

## Быстрый просмотр paper-результата

```powershell
.\3bar\paper_summary.ps1
```

Скрипт показывает:

- активную paper-позицию из `state/live_state.json`
- число уже закрытых paper-сделок
- wins / losses / win rate
- суммарный `gross_pnl` и `net_pnl`
- последние 10 закрытых paper-сделок

## Настройки в .env

Главные поля:

- `BINGX_API_KEY`
- `BINGX_SECRET_KEY`
- `PAPER`
- `SYMBOL`
- `INITIAL_CAPITAL`
- `RISK_PERCENT`
- `POLL_SECONDS`
- `HEARTBEAT_MINUTES`
- `TRAIL_ACTIVATION_R`
- `TRAIL_LOOKBACK_BARS`
- `MAX_HOLD_BARS`

## Безопасность

- по умолчанию запуск идет в paper-режиме
- real-режим требует `--confirm-live`
- daemon не запускается второй копией, если уже есть lock-файл
