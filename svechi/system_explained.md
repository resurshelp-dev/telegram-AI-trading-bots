# Svechi System

## Что это за система

`Svechi` это автоматическая ETH-USDT система на свечных паттернах.

Она работает по историческим и актуальным `5m` свечам, сама строит старшие таймфреймы, ищет сигналы, считает результаты и отправляет Telegram-сводку.

Главный файл настроек системы:

- `svechi.env`

## Какие таймфреймы использует

Базовые данные:

- `5m`

Из них автоматически строятся:

- `10m`
- `15m`
- `20m`
- `30m`
- `45m`
- `1h`

В финальной торговой системе реально используются эти сигнальные блоки:

- `15m shooting_star short`
- `20m morning_star long`
- `30m bearish_engulfing short`
- `10m hanging_man short`

## Как система входит в рынок

Для каждого сигнала:

- паттерн должен быть найден на своём таймфрейме
- если для модуля включён фильтр объёма, объём должен быть не ниже `median volume 20`
- вход делается на `open` следующей свечи после сигнала

Если в этот момент уже есть активная сделка из системы, новый сигнал пропускается.

## Как система выходит

Каждый модуль использует только:

- жёсткий `ATR stop`
- `time stop`

Фиксированного тейка нет.
Безубытка нет.
Трейлинга нет.

Это сделано специально: на разных тестовых месяцах эти усложнения не дали устойчивого улучшения без риска переоптимизации.

## Точки входа и выхода по модулям

### `15m shooting_star short`

- вход: на открытии следующей `15m` свечи после сигнала
- стоп: `0.8 ATR`
- удержание: `12` баров `15m`
- роль: основной short-модуль на локальном истощении роста

### `20m morning_star long`

- вход: на открытии следующей `20m` свечи после сигнала
- стоп: `0.8 ATR`
- удержание: `12` баров `20m`
- роль: основной long-модуль на локальном развороте вниз-вверх

### `30m bearish_engulfing short`

- вход: на открытии следующей `30m` свечи после сигнала
- стоп: `1.2 ATR`
- удержание: `8` баров `30m`
- дополнительно: `min_body_ratio=0.2`
- дополнительно: `volume_filter=on`
- роль: более сильный short-модуль для подтверждённого разворота

### `10m hanging_man short`

- вход: на открытии следующей `10m` свечи после сигнала
- стоп: `0.8 ATR`
- удержание: `12` баров `10m`
- дополнительно: `volume_filter=on`
- роль: быстрый short-модуль для добора импульсных сигналов

## Что система использует друг у друга

Модули не подтверждают друг друга напрямую.

Но они используют общие правила:

- один общий поток сигналов
- запрет на наложение позиций
- единый расчёт ATR
- единые комиссии
- единый журнал сделок

То есть это не набор независимых ботов, а одна комбинированная система с общим риск-контуром.
