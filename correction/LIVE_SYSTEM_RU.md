# Финальная live-сборка

## Что сделано

- Собрана одна автономная торговая система для `ETH-USDT`.
- Основной запуск один: `launch_system_window.cmd`.
- Система работает в отдельном окне, в режиме daemon, без ручного участия.
- Добавлены Telegram-оповещения:
  - старт системы
  - найден сигнал
  - результат исполнения
  - ошибка
  - heartbeat
- Добавлена защита от конфликтов:
  - только один экземпляр системы через `state/correction_system.lock`
  - если по `ETH-USDT` уже есть позиция или открытые ордера, новый вход не открывается

## Какие стратегии входят в live

- `basic correction`
- `trend = profit_max_locked`

Это текущая боевая связка по умолчанию.

## Полноценный paper-режим

- Режим переключается только через `.env`
- для paper ставится `PAPER=true`
- для real ставится `PAPER=false`
- запуск один и тот же: `launch_system_window.cmd`

Что делает paper-режим:

- открывает виртуальную сделку по сигналу
- ведёт одну открытую paper-позицию
- отслеживает `TP1`, `TP2`, `stop`, `time stop`
- считает `net pnl`, `R-multiple`, win rate и текущий виртуальный капитал
- пишет отчёты в:
  - `state/paper_state.json`
  - `reports/paper_trading/paper_report.json`
  - `reports/paper_trading/paper_trades.jsonl`

## Что сделано со sweep

- Отдельная стратегия `Liquidity Sweep + Reclaim` реализована и протестирована.
- Лучший профиль: `sweep_15m_round_reclaim_combo`.
- На мартовском окне он улучшал результат.
- На январских окнах он ухудшал результат.

Вывод:

- `sweep` не включён в live по умолчанию.
- Он оставлен как исследовательский модуль.
- В `.env` он управляется через `SWEEP_PROFILE`.
- Без отдельного regime-filter безопасный default: `SWEEP_PROFILE=off`.

## Как работает механика

### Basic correction

- Ищет коррекционные входы по уже собранной модели уровней и подтверждений.
- Вход даётся только когда цена находится в рабочей зоне.
- После входа автоматически ставятся защитные ордера.

### Trend

- Использует профиль `profit_max_locked`.
- Берёт только самые качественные trend-pullback сигналы.
- Работает как второй независимый источник сигналов.

### Supervisor

- Собирает сигналы из всех активных модулей.
- Не даёт двум сделкам пересекаться во времени.
- Выбирает лучший доступный сигнал по времени, приоритету и качеству.

## Полная автоматизация

- Запуск: двойной клик по `launch_system_window.cmd`
- Внутри запускается `run_system.ps1`
- Далее система непрерывно:
  - сканирует рынок
  - выбирает лучший сигнал
  - проверяет конфликты
  - открывает сделку
  - ставит защиту
  - пишет состояние и лог
  - шлёт уведомления в Telegram

## Где настраивается риск

Риск теперь настраивается прямо в `.env`, там же, где ключи BingX и Telegram.

Главные параметры:

- `INITIAL_CAPITAL=10000`
- `RISK_PERCENT=1`
- `TREND_PROFILE=profit_max_locked`
- `SWEEP_PROFILE=off`
- `POLL_SECONDS=60`
- `HEARTBEAT_MINUTES=30`

Практически:

- если нужен риск `0.5%`, ставишь `RISK_PERCENT=0.5`
- если нужен риск `2%`, ставишь `RISK_PERCENT=2`

## Главные файлы

- `launch_system_window.cmd` — единственный файл, который нужно запускать
- `run_system.ps1` — внутренний технический запуск, его обычно трогать не нужно
- `.env` — ключи и настройки риска
